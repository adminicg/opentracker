Qt 2 PIP Transformation

The qt2pip.xsl style sheet transforms a forms data file of Qt Designer 2.0 (part of Qt 3.0.x) into a PIP sheet for Studierstube applications. It uses as much information as possible from the widgets defined in the form file such as size and location, label or image, fore- and background color. It transforms the following widgets

Qt Widget Studierstube Widget
QDialog PIP Sheet (as a SoPipSheetKit node)
QTabWidget
 underlying QWidgets (the tabs) will group their children into individual separators to get Studierstube tabs
QButtonGroup SoGroup with all children widgets as children of the SoGroup
QRadioButton SoRadioButton
QPushButton SoPushButton
QCheckBox SoToggleButton
QSlider SoSlider
QLabel Either a text label or an image on the PIP
QListBox
 SoListBox with items as values.

The following properties of all widgets are used :

  • x,y - position on the PIP sheet
  • width, height - dimension off the widget as width and depth
  • name - DEF name of the widget
  • text - text label on the widget
  • image - an image on the widget
  • paletteForegroundColor - color of the text label
  • paletteBackgroundColor - color of the widget geometry

Other parameters of the widgets need to be set in the xsl script as parameters or global variables :

  • height - the height of widgets
  • default text style, family or font name
  • geometry - there is a set of variables for each widget type which are substituted into the geometry parts of widgets. In the default settings they all reuse the same geometry, therefore all widgets will look the same.
  • highlight default color - the default highlight color to use, if no background color is specified.
  • testmode - true or false, default is false. If TRUE it will leave out field connections and generate a simple application wrapper to allow quick testing of pip designs.

Some notes :

  • The PipSheet will be scaled so that the larger dimension will have a length of 1. All dimensions of sub widgets etc. are scaled accordingly. The global height variable is not affected by this.
  • To use images on widgets or image labels, you need to do the following in Qt Designer :
    1. Set the form properties to use a function to load images (! not inline images !)
    2. Set the pixmap parameter to the path and filename of the image you want to use. You will not get a preview of the image unfortunatly.
  • A label with a pixmap property defined will always be transformed to an image label and ignore the text property. If you want to use the text property make sure to delete the pixmap property in the designer.
  • tab sheets are autoscaled by the Studierstube individually, the computation of widget dimensions is done for the whole dialog ! Therefore individual tabs can be scaled incorrectly. Make sure you have widgets covering the same extensions in each tab, then you should be fine.
  • The stylesheet now uses new button geometry that is somewhat simpler than the old one and is compatible with the SoListBox default style.
  • Text on buttons will always be look ok, because it will only be uniformily scaled. However, if the string is too long, it can protrude over the ends of the button !
  • Text in labels is now horizontally aligned with the geometry by specifying either AlignLeft, AlignCenter or AlignRight. Every other alignment will default to centered. The text is also always uniformily scaled to fit with its height. Therefore it can also protrude over the label area if too long.
  • Text labels can also be vertical by specifying a tooltip text of 'vertical'.
  • If a background color on a text label with text is specified, a background quad will be created carrying that color. This also makes text only labels count in bounding box calculations.
  • All in all, text is more sane now. Any label or button with the same widget height should result in the same text size, for vertical labels the width counts.
  • whatsThis field is used to define additional data for the widgets and other parts :
    • QLabels can specify a nodename.fieldname to have a route connection written out for their text field. This allows to use text labels as message displays in a simple manner
    • QTabWidget (for tabbed sheets) can specify a space-separated list of file names that go into the tabTextures field to specify the button textures for individual tabs.
  • entries of QListBoxes will be written to the inventor file as is. This allows you to specify more complex things in here !
  • Sliders only use background color for the static geometry and foreground color for knob geometry with a highlighted version.

Studierstube Development on Unix

Getting Started

  1. Check out Studierstube from the cvs server
  2. Change to the Studierstube base directory $(STBROOT), e. g., $(HOME)/studierstube/ if you checked out from your home directory
  3. Make sure the link Config.site points to the right OS-dependent config file, i. e., either Config.Irix if you are running Irix, or Config.Linux if you are running Linux.
  4. Edit the settings in Config.global and Config.[YOUR_UNIX] to reflect your particular development environment. The config file contains instructions what needs to be done.
  5. Execute "make all" to build the Studierstube software

Creating a new application

  1. When you create a new application, you must make sure it will run under both Unix and Win32 OS families. Therefore you need to also to consult the instructions for Win32.
  2. Create a Makefile for Unix in your application’s source directory. Use $(STBROOT)/SAMPLE_MAKEFILE as a template
  3. Enter your application into the list of application projects in $(STBROOT)/src/apps/Makefile
  4. Compile, test!

How to get Studierstube work under Linux: Compiling Studierstube under Linux

(tested with SUSE 7.1, Kernel 2.4)

In order to run Studierstube you must compile SGI OpenInventor (OIV) first. After having compiled OIV, compiling the studierstube is no major issue.
Before you start compiling OIV you must have Mesa (OpenGL for Linux) installed plus all header files. You also need the LessTif header files (=Xm header files)

Inventor:
Get SGI OpenInventor from ftp://oss.sgi.com/projects/inventor/download/
We used inventor-2.1.5-6.src.tar.gz
Uncompress it and compile it. It didn't fully compile on our system (applications were not compiled) but the libInventor.so was created and libInventorSoXt.so was created too. You don't need more. Install the files on your system.
If you want to compile the applications like ivcat, ivview,...... you can do this manually. It should work.

Studierstube:
Check out the latest Studierstube from the cvs server (check the CVS documentation for instructions about how to check out under Linux).
Edit studierstube/Config.global and studierstube/Config.Linux to make sure that you build Studierstube for SGI OpenInventor!
Make
If it compiled successfully (it is supposed to do so), run workspace and have fun!
 

Known problems:

  • We did not compile any font library, therefore Text did not work in our examples.
  • We could not use the Gesture Library by Oliver Bimber. It does not compile correctly. 
    Comment it out (this avoids compilation of IMGR) by removing "#define GRT_IMGR" in studierstube/src/stbapi/misc/GRT_defs.h
    There will soon be a compiler option in Config.Irix to avoid conflicts and to avoid fiddling around with the GRT_defs.h file!

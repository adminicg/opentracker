Compiling and running studierstube on the Onyx (merkur.edvz.uni-linz.ac.at)

Our account name on the Onyx merkur.edvz.uni-linz.ac.at which is running the CAVE in Linz is k2e5070. The account belongs to Hannes Kaufmann (kaufmann@ims.tuwien.ac.at) who is responsible for it. You can get details by contacting Hannes Kaufmann.

General comment: GNU make can be found in ~k2e5070 in the directory /make..../

Compiling QT:

[Qt downloaded from www.trolltech.com]

QT must be compiled as a 32 Bit application (by default the makefiles are configured for a 64 bit application). Otherwise SoQt (used in connection with our OpenSource Inventor) cannot be compiled correctly.
Add GIF and JPEG support when configuring the Qt makefiles by using the following command
./configure -platform irix-n32 -gif -system-jpeg
If you want to compile QT as a multi-threaded application, use the additional flag -thread. I did not compile it multithreaded because the library will get a different name and maybe SoQt does not compile afterwards. If you need multi threading, you have to compile the whole Qt again.
Enabling sound system, MNG I/O support and anti-aliased fonts does not work - compilation will stop!

Compiling OpenSource Inventor:

[Open Source Inventor downloaded from www.cg.tuwien.ac.at/hostings/openinventor]

To help you avoid the troubles that I had, there are 2 things that must be done before compiling Inventor like described by Gerd Hesina in his README.

1.) When compiling Inventor, the whole NURBS nodes and stuff doesn't get compiled automatically. The Inventor compilation continues, but when trying to start workspace or any other inventor application you get an error message from libInventor about the NURBS stuff. To solve this problem, compile the whole NURBS stuff manually by going to
/home/k2e02/k2e5070/inventor/build/inventor/lib/database/src/so/nodes/nurbs
and execute GNUmake. This will compile all NURBS related files in this directory and the subdirectories.

2.) When compiling Inventor, readGIF.c cannot be compiled. This is due to some strict compiler (they probably didn't update their compiler for some time). Inventor compilation continues but you'll get the error message when executing workspace later on. While compiling you get the error message
"../../../../../inventor.h", line 2: error(1166): expected a declaration
Solution: do not include inventor.h in readGIF.h (uncomment /* #include <inventor.h> */). Another solution would probably be to add a #define IRIX in inventor.h
Then compile Inventor (or all files in this directory manually by executing make) and everything will work.

compile the whole Inventor package

Compiling Studierstube:

[got by cvs - see cvs description about how to check out Studierstube]

Errors when compiling on the Onyx:
These 3 errors should not occur in the newest version of the stb!!!
1.) isdriver.h is included in TrackerIS.h, but the path to the .h file of this driver is wrong. The correct path (code snippet) is
extern "C" {
#include <../../../extras/intersense/isdriver.h>
}

2.) When linking workspace you get an error when linking the Intersense driver.
Solution: go in isense.h down to the bottom of the file where the procedures ISD_OpenTracker and many others are listed. They are already in " extern "C"{ " brackets but " extern "C"{ " is uncommented. Activate " extern "C"{ " and compile it again and linking should work this time.

3.) in StbAppManager.cxx the file zpthread.h is included. When compiling the file this leads to an "u_int64_t undefined error" and stops the compilation. Remove zpthread.h from the StbAppManager.cxx (uncomment it) and the file will compile.

compile studierstube

now starting workspace should work!

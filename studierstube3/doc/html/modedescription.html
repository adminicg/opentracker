Display modes:

General options:

position of the window (in pixels) for up to four viewers:

option: --xoffset[234] 0

x-coordinate of the upper left corner of the window

option: --yoffset[234] 0

y-coordinate of the upper left corner of the window

note: the upper left corner of the screen has the coordinates x = 0, y = 0

size of the window (in pixels) for up to four viewers:

option: --width[234] 640

width of the window

option: --height[234] 480

height of the window

note: the default values for position and size are set for usage with display mode LINE using iglasses

Mono:

option: --display MONO (default)

creates an examiner-viewer style window with widgets

note: only the left camera will be used for rendering, but the camera-file specified with the option ‘—camera’ must always contain two cameras (see also camera control modes for further details)

Line interleaved:

option: --display LINE

use this mode for iglasses
creates a window without a border
the frames for left and right eye are rendered in the same buffer, where the even lines are used for the left eye and the odd lines for the right eye

note: always use an even number of lines (window height), otherwise the left and right eye will be rendered in the wrong buffer

Field interleaved:

option: --display FIELD

use this mode for shutter glasses
creates a window without a border, this mode uses quadbuffering or STR_TOP stereo rendering

option: --splitstereo

use STR_TOP mode

note: because this display mode stretches the desktop in vertical direction, the window size height should be the half of the desktop height and the window width should equal that of the desktop

option —visualdevice 1

choose visual device (only needed for quad-buffer rendering)

possible values:
0 … not supported (no stereo rendering)
1 … standard visual device supporting stereo (stereo rendering: works for Impact graphics adapter and Intense 3D Wildcat 4000)

note: for other graphics hardware a new StbVisual class may be needed

Dual channel:

option: --display DUAL

use this mode for barco doubleprojector with passive polarisation
this display mode also uses only one window (without border), but two viewports.

Per default the window is split vertically, so that the frame for the left eye is rendered in the left half of the window. When the

option: --splitstereo

is used the window is split horizontally, so that the frame for the left eye is rendered in the upper half of the window.
The window position and size must be set accordingly.

 

Camera control modes:

 

General options:

option: --camera defaultCamera.iv

specifies the camera-file to search for cameras
The camera-file must always contain two SoOffAxisCameras, the first camera found in the file will be used as camera for the left eye and the second for the right eye.

 

Desktop control:

option: --camcontrol DESKTOP (default)

this control mode works like an Examiner-Viewer, hence the user controls the camera with the mouse or keyboard.

additonal key commands (to Examiner-Viewer viewing control):

home … home position
insert … save home position
end …... view all

option: --headlight

turns the headlight on

note: this option only works correctly for desktop control mode

ad option: --camera

The desktop control mode calculates the middle eye position and orientation for a "middle eye camera", hence even if the display mode is mono, there need to be two valid cameras in the specified file
(tip: in the case of mono rendering just use the same camera for left and right eye)

camera example: (useful for display mode mono)

#Inventor V2.1 ascii
Info {
	string "this is the camera for the LEFT eye"
}
SoOffAxisCamera {
	viewportMapping ADJUST_CAMERA
	nearDistance 0.01
	farDistance 100
	focalDistance 2
	size 0.35 0.27
	eyepointPosition 0 0 0.5
} 

Info {
	string "this is the camera for the RIGHT eye"
}
SoOffAxisCamera {
	viewportMapping ADJUST_CAMERA
	nearDistance 0.01
	farDistance 100
	focalDistance 2
	size 0.35 0.27
	eyepointPosition 0 0 0.5
}

 

Tracked viewpoint:

option: --camcontrol VIEWPOINT

This control mode tracks the head (left and right eye) but the projection planes of the cameras are fixed.

option: --stereokit myKit.iv

specifies the file which contains a SoStereoCameraKit

Only the fields eyeOffsetLeft and eyeOffsetRight need to be specified.
eyeOffsetLeft: the offset between head tracker and the left eye in head tracker space
eyeOffsetRight: the offset between head tracker and the right eye in head tracker space

note: the file must not contain any other node, see example below

stereokit example:

#Inventor V2.1 ascii
SoStereoCameraKit {
	eyeOffsetLeft -0.0305979 0.147538 0.0151514
	eyeOffsetRight 0.0373777 0.148288 0.0177628
}

camera example:

#Inventor V2.1 ascii
Info {
	string "this is the camera for the LEFT eye"
}
SoOffAxisCamera {
	viewportMapping LEAVE_ALONE
	nearDistance 0.16
	farDistance 5
	focalDistance 2
	size 0.35 0.27
	position -0.26 -0.58 -0.95
	orientation 0 1 0 -1.574
} 

Info {
	string "this is the camera for the RIGHT eye"
}
SoOffAxisCamera {
	viewportMapping LEAVE_ALONE
	nearDistance 0.16
	farDistance 5
	focalDistance 2
	size 0.35 0.27
	position -0.26 -0.58 -0.95
	orientation 0 1 0 -1.574
}
note: for tracked viewpoint control the parameters for the projection plane should be the same for the left and right eye (position, orientation, size), whereas the eyepointPosition has no effect (because it is updated permanently using the tracker data and the specified offsets)

 

Tracked display:

option: --camcontrol DISPLAY

The tracked display control mode uses headtracking but contrary to the tracked viewpoint control mode moves the whole camera (projection plane and eyepoint), hence the name tracked display.
The camera parameters are relative to the head tracker (in head tracker space).

camera example: (for iglasses)

#Inventor V2.1 ascii
Info {
	string "this is the camera for the LEFT eye"
}
SoOffAxisCamera {
	viewportMapping LEAVE_ALONE
	position -0.037045 0.168842 0.236637
	orientation 0.999133 -0.00585241 0.0412087 2.99499
	nearDistance 0.16
	farDistance 5
	focalDistance 2
	size 0.1 0.075
	eyepointPosition -0.0305979 0.147538 0.0151514
} 

Info {
	string "this is the camera for the RIGHT eye"
}
SoOffAxisCamera {
	viewportMapping LEAVE_ALONE
	position 0.0252067 0.170702 0.245833
	orientation 0.999225 0.00158786 0.0393301 3.02502
	nearDistance 0.16
	farDistance 5
	focalDistance 2
	size 0.1 0.075
	eyepointPosition 0.0373777 0.148288 0.0177628
}
note: the easiest way to determine the parameters is to use the calibration application located in the directory bin/apps/calibration.
This application creates a file which can be directly used as parameter for the --camera option. (See LINK!!!)

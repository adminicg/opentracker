Desktop AR Setup

This page describes the setup for a desktop based fish-tank-VR Studierstube configuration based on ARToolkit tracking of PIP and pen. This setup provides you with a video overlay AR setup where you see the view of the camera overlayed with the input devices. You can use cardboard tools to interact with PIP and pen and the applications. As button the keyboard input is used. So space is the primary button. To be able to click buttons on the PIP, it will not vanish even if you move the real cardboard PIP away. This gives you a free hand to use the keyboard to click.

The following prerequisits are necessary to make it work.

  1. OpenTracker should be compiled with ARToolkit and work with your webcam. This camera is used to track the PIP and pen replacements.

  2. To build your own PIP and pen, use the cut-out template tools.pdf. Print it and glue it to a thin card board then cut out the tools along the lines indicated.

  3. The tip of the pen is the corner of the ARToolkit marker that lies opposite to the long handle. Currently only the pi symbol is tracked by the given configuration file. The task of configuring the backside is left as an exercise to the reader.

  4. In studierstube\bin\workspace start desktop.bat. This starts your fish-tank-VR Studierstube as described above.

  5. That's it !

The following files are part of the configuration

  • desktop.xml - OpenTracker configuration file, describing the ARToolkit Marker,currently only Pi and Lambda, not Kappa (the backside of the pen).
  • data/webcam.dat - camera calibration file of the webcam, replace this with your own calibration file
  • data/pi.patt, data/kappa.patt, data/lambda.patt - pattern files for ARToolkit
  • desktopUserKit.iv - Userconfiguration for Studierstube, defines the stations and enables the video background
  • desktopCamKit.iv - modified camera describing the web camera model
  • desktop.bat - a simple batch file starting Studierstube with the above configuration files

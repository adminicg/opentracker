"SAPI Speech Support for Opentracker"

OVERVIEW

The new SAPI Speech Support for Opentracker represents a flexible and extensible implementation.
Speech events are fired by a Speech Recognition module, in the Opentracker event handling mechanism.

Speech features:

  • Speech Recognition in several languages and products
  • Text To Speech in several languages and voices
  • Full configurable

Speech support for Studierstube:

  • Widget and Gesture Speech support
  • Raw SR/TTS support

 

Speech Configuration in Opentracker:

The following application specific xml configuration show some examples of possible useful usage of the speech component in opentracker:

<OpenTracker>
            <configuration>
                        <SpeechRecoConfig language="english">
                                   <CommandSet id="CmdSet1">
                                               <Command id="1234" name="one"/>
                                               <Command id="1235" name="two"/>
                                               <Command id="1236" name="six"/>
                                               <Command id="1237" name="seven"/>
                                   </CommandSet>
                                   <CommandSet id="CmdSet2">
                                               <Command id="1234" name="count"/>
                                               <Command id="1235" name="get"/>
                                               <Command id="1236" name="goto"/>
                                               <Command id="1237" name="compile"/>
                                               <Command id="1238" name="up"/>
                                               <Command id="1239" name="down"/>
                                   </CommandSet>
                        </SpeechRecoConfig>
            </configuration>
            <StbSink station="0" event="on">
                        <StbKeyboardSource number="0"/>
            </StbSink>
            <StbSink station="6" event="on">
                        <SpeechRecoSource name="Wordset" set="CmdSet2"/>
            </StbSink>
            <StbSink station="7" event="on">
                        <SpeechRecoSource name="Numberset" set="CmdSet1"/>
            </StbSink>
</OpenTracker>

Explanation of speech specific tags:

  • <SpeechRecoConfig>      select the speech of the SR component.
  • <CommandSet>              define a set of speech commmand.
  • <Command>                   define a single speech command.
  • <SpeechRecoSource>     build a speech event firing source node with a specific command set

 

Speech usage in Studierstube:

Widgets:

Gestures (see SoGesture) changed to Nodes. They could now be defined in .iv files, and applied and configured
dynamic for nearly all widgets. Gestures now could be configured to listen only on specific stations, because they
listened on all station and interpreted speech events as incorrect translation events.

There is a new Gesture, namely SoSpeechGesture.
The gesture could simulate begin and end speech events for button press and release actions.

The following Widgets were made speech able and configurable:

  • SoButtonKit
  • SoPushButton
  • SoToggleButton
  • SoRadioButton
  • SoSpeechSlider (wrapper on SoSlider)

Example usage of a speech enabled SoPushButton in a Studierstube application: (configuration via .iv file)

            DEF RESET_BUTTON SoPushButton
            {
                        width 0.1
                        height 0.05
                        depth 0.1
                        hysteresisBoxScale 1.0

                        actionEnterGestures
                        [
                                   SoEnterGesture{ stations [ 0 ] blocked FALSE }
                        ]
                        actionExitGestures
                        [
                                   SoExitGesture{ stations [ 0 ] blocked FALSE }
                        ]
                        actionStartGestures
                        [
                                   SoBeginSpeechGesture
                                   {
                                               stations [ 6 ] blocked FALSE
                                               SpeechCommands [ "count", "get" ]
                                   },
                                   SoButtonPressGesture{ buttonNumber 0 stations [ 0 ] blocked FALSE }
                        ]
                        actionEndGestures
                        [
                                   SoEndSpeechGesture
                                   {
                                               stations [ 6 ] blocked FALSE
                                               SpeechCommands [ "count", "get" ]
                                   },
                                   SoButtonReleaseGesture{ buttonNumber 0 stations [ 0 ] blocked FALSE }
                        ]
            }

 

Custom Speech usage:

To use SR/TTS in your application, do the following steps:

  • Link "opentrackerd.lib" to the application.
  • Define "USE_OPENTRACKER" and "USE_SAPISPEECH" as preprocessor directives.
  • Add OT Include Path to project (C/C++ Options).                  /I "$(OTROOT)\src"
  • Add OT Lib Path to project (Link Options).                             /libpath:"$(OTROOT)\lib"
  • #include <stbapi/tracker/TrackerOT.h>
  • #include <input/SpeechModule.h>
  • config speech commands via xml file
  • look for the appropriate event on the correct station

Code example of use in the windowEventCB procedure:

  if(messageId == WM_EVENT)
  {
    // event must be on the correct station, only take the starting speech event, and the button change event
    if(event->getStation() == 6 && event->getType() != So3DEvent::ET_BUTTON0_EVENT && event->getButton(0) == TRUE)
    {
      TrackerOT *tracker = (TrackerOT*)Stb3DEventGenerator::getTracker();
      Context *context = tracker->getContext();
      SpeechModule *SRModule = (SpeechModule*)context->getModule("SpeechRecoConfig");
      assert(SRModule != NULL);

      DWORD CommandId = (DWORD)event->getTranslation()[0];
      DWORD SpeechSetId = (DWORD)event->getTranslation()[1];

      // retrieve the appropriate command string
      std::string Command;
      SRModule->GetCommand(CommandId, SpeechSetId, Command);

      // example of how to speak a string
      SRModule->Speak(Command.c_str());
    }
  }

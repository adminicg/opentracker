Textures

Textures can be applied to shapes using the SoTexture2 node. The texture can be loaded from a file in the common graphics formats gif or jpg. For applying the texture to a shape, texture coordinates have to be defined in a seperate TextureCoordinate2 node.

The following example loads an image from a file and applies it to a square-shaped IndexedFaceSet: 

Coordinate3 {          # this sets the current coordinates for all subsequent nodes
    point [-0.5 -0.5 0, 0.5 -0.5 0, 0.5 0.5 0, -0.5 0.5 0]
}
Texture2 {             # loads the texture from file
    filename "foo.jpg"
    model DECAL
}
TextureCoordinate2 {   # assigns texture coordinates to the vertices defined above
    point [0 0, 1 0, 1 1, 0 1]
}
IndexedFaceSet {       # constructs a textured face from the above information
    coordIndex [0, 1, 2, 3, -1]
    textureCoordIndex [0, 1, 2, 3, -1]
}

Lights

Lights can be added by the scene by simply adding a light node. The position of the light is determined by the transformation nodes that are traversed before it. Lights are stored in the traversal state and can therefore be applied to a part of the scene graph by using separators.

Cameras

Like lights, cameras are ordinary scene objects that can be added to the scene graph. If a camera is traversed, all subsequent content will be rendered using this camera! This allows to render parts of the scene with a different camera, to create heads-up displays or overlay graphics. Of course, Separators can be used to constrain the effect of a camera to a well-defined part of the scene.

Studierstube provides its own camera node - SoOffAxisCamera* - that allows flexible control over all parameters need for rendering augmented reality applications.
